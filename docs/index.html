Alfalfa Project:

  • Parse vcf files, extract readcounts, filter out all variants with more than two alleles. This file contains some EDA on the VCF files. But mostly, this code gets the read-count matrices.
  • Initial fits for some SNPs. These data seem messier than the sweet potato data. Though, there seems to be much less sequencing error. Bias is still a possible problem for some SNPs.
  • Tons of raw plots Tons of raw plots. I am trying to see features of the data.
  • Try the bias corrected updog on the alfalfa data. Some things look good, some things look bad.
  • Motivate Prior on sequencing error. I think a reasonable prior is Beta(mean = 0.005, od = 0.02). This is motivated by the sequencing error rates that the Blischak paper assumes.
  • Better prior on sequencing errorr rate. Here, I just use a logit-normal. This is better because the beta has an infinite spike at 0 for reasonable priors. The logit normal works much nicer in practice and is intuitive.
  • Motivate Prior on bias parameter. I think a reasonable prior is N(0, 1) on the log-bias parameter. Values of -3 and 3 are super extreme and I would consider to be extremely poorly behaved SNPs that I would probably throw out if we ever saw them.
